## 整体需求说明
本项目是一个类似苹果Reminder的功能，底层代码使用Markdown格式，但是会在Markdown格式的基础上进行修改。

1. 用户启动软件后，首先需要在本地选择一个目录，该目录作为Reminder的工作目录，这样做的目的是便于Reminder的多端同步和备份
2. 在系统设置中添加一个设置项，叫做工作目录。用户可以修改工作目录，并读取该工作目录下的Reminder列表
3. 工作目录下保存多个TXT文件，每个TXT文件对应一个待办事项列表
4. 在工作目录下创建配置文件config.yaml，用于保存系统的所有配置

## 文件解析说明
每一个任务列表的底层是一个TXT文本，每个任务被分为上下两行，上行为JSON，记录该任务的相关信息，下行为text，记录该任务的具体内容。
```json
{
    "CreateTime": "2026-01-01 12:30:00",
    "Deadline": "2026-01-01 12:30:00",
    "Level": 1,
    "Status": "TODO",
}
```
| 字段 | 数据类型 | 说明 |
| ------- | ------- | ------- |
| CreateTime  | Time stamp | 该事项被记录的时间，用户输入时由系统自动创建  |
| Deadline  | Time stamp  | 该事件的截止时间  |
| Level | Int | 大于等于一的整数，Level为一表示该事件为独立事件，Level为二表示该事件为上一个Level为1的事件的子事件，以此类推 |
| Status | 字符串枚举值 | 枚举值包括TODO（待处理）、DONE（已完成）和CANCELED（已取消） |

### 窗口整体设计

- 窗口整体 UI 布局参考 macOS 系统设置：左侧为主菜单列表，右侧显示当前菜单对应的设置内容。
- 设置窗口默认尺寸为 720 × 520 pt。窗口宽度固定且不可调整；高度可由用户调整，最小高度为 480 pt。
- 左侧主菜单默认宽度为 176 pt，并允许系统在 160–192 pt 范围内适配；右侧设置内容的最小宽度为 544 pt。
- 设置窗口不显示系统自动生成的 `隐藏边栏`（`Hide Sidebar`）按钮。
- 设置窗口标题为 `设置`（`Settings`）。按 `ESC` 关闭设置窗口。
- 左侧主菜单依次显示 `通用`（`General`）、`行为`（`Action`）、`任务优先级`（`Priority`）和 `番茄任务`（`Pomodoro Task`）；窗口打开时默认选中 `通用`（`General`）。
- 四个主菜单分别使用齿轮、开关、旗帜和计时器图标。点击左侧菜单后，右侧只显示当前菜单对应的内容，并同步更新右侧页面标题。
- 当前原型切换语言后会立即更新设置窗口的左侧菜单、右侧标题、控件名称、提示和弹窗。正式版本还需覆盖菜单、右键菜单、状态名称及错误信息；用户创建的任务文本、列表名称和自定义优先级名称不自动翻译。

### `通用`（`General`）

#### 语言与外观

1. `语言/Language`（`语言/Language`）
   - 使用宽度为 170 pt 的双段控件；左侧固定显示 `中文`（`Chinese`），右侧固定显示 `English`（`English`），不随当前界面语言翻译。该控件保持紧凑，不与颜色模式控件等宽。
   - 首次启动时根据系统首选语言设置默认值；用户后续选择会持久化到 `config.yaml`。
   - 当前原型切换后立即更新设置窗口中的系统文案；正式版本再扩展到应用内全部系统文案。
2. `颜色模式`（`Color Mode`）
   - 使用分段控件，依次显示 `浅色`（`Light`）、`深色`（`Dark`）和 `跟随系统`（`System`）。

#### 工作目录

1. `工作目录`（`Work Directory`）显示当前工作目录路径，路径可选择和复制。
2. 下方提供四个按钮：
   - `选择工作目录`（`Select`）：打开系统文件夹选择器。若已有工作目录，选择新目录后必须显示二次确认；按钮为 `放弃`（`Cancel`）与 `确认`（`Confirm`），其中 `确认`（`Confirm`）使用红色破坏性样式。
   - `打开目录`（`Open`）：在系统文件管理器中打开当前工作目录。
   - `config.yaml`（`config.yaml`）：使用系统默认文本编辑器打开配置文件。
   - `刷新`（`Refresh`）：重新读取工作目录中的列表和配置。
   - `选择工作目录`（`Select`）使用带加号的文件夹图标，`打开目录`（`Open`）使用普通文件夹图标，避免两个操作使用相同图标。
   - 四个按钮之间使用 12 pt 间距；每个按钮内部的图标与文字使用 2 pt 紧凑间距。`刷新`（`Refresh`）使用与其他按钮一致的线性图标风格。

### `行为`（`Action`）

该页面是左侧主菜单的第二项，位于 `通用`（`General`）之后、`任务优先级`（`Priority`）之前，用于集中展示复制和任务显示行为设置。

1. `显示任务序号`（`Show Task Numbers`）控制任务列表是否显示序号。
2. `复制时包含任务序号`（`Include Task Numbers When Copying`）紧接在其下方并向右缩进 22 pt，作为 `显示任务序号`（`Show Task Numbers`）的子选项；当父选项关闭时，此选项不可用。
3. `复制时播放音效`（`Play Sound When Copying`）控制复制任务或任务文本时是否播放音效；正式接入业务逻辑后，开启该选项时复制操作会播放提示音。
4. `完成任务隐藏延迟`（`Completed Task Hide Delay`）使用宽度为 64 pt 的数值输入框，范围为 0–5000 毫秒；中文界面显示单位 `毫秒`（`Milliseconds`），英文界面显示 `ms`（`ms`）。它控制已完成任务在需要被过滤隐藏前保留在列表中的时长。

### `任务优先级`（`Priority`）

本 section 同时显示系统内置优先级和用户自定义优先级。每一行从左到右包含以下内容：

1. 圆形色块：自定义优先级可点击打开颜色选择器。颜色选择器提供 30 种预设颜色和自定义颜色；内置优先级颜色不可修改。色块与后方 `粗体`（`Bold`）选项之间使用 12 pt 间距。
2. 文字样式：`粗体`（`Bold`）、`下划线`（`Underline`）和 `斜体`（`Italic`）。内置和自定义优先级都允许修改这三项。
3. 优先级名称：文字右对齐。内置优先级名称不可修改；自定义优先级名称可修改，且不能为空。若输入为空，立即阻止保存并显示 `名称不能为空`（`Name Is Required`）。
4. 删除按钮：内置优先级显示灰色垃圾桶，鼠标悬停时提示 `内置优先级不可删除`（`Built-in priorities cannot be deleted`）；自定义优先级显示垃圾桶，常态为黑色，鼠标悬停时变为红色。优先级名称与垃圾桶之间使用 8 pt 最小间距；删除前显示二次确认。
5. `添加优先级`（`Add Priority`）位于全部优先级下方。新行复制当前最下方优先级的颜色和文字样式，名称为 `自定义优先级`（`Custom Priority`）；创建后自动聚焦名称输入框并全选默认名称。

每行固定为 28 pt 高；优先级名称区域宽度为 105 pt。当前原型支持切换三种文字样式、修改自定义颜色和名称、添加自定义优先级，以及通过确认弹窗删除自定义优先级。

优先级的名称、颜色和文字样式等设置修改均支持 `Command Z`（`Command Z`）撤销上一次修改。

### `番茄任务`（`Pomodoro Task`）

本 section 同时显示内置时长和用户自定义时长。每一行从左到右包含：

1. 计时图标。
2. 时长。格式使用 `__ 小时 __ 分钟`（`__ Hours __ Minutes`）。
3. `编辑时长`（`Edit Duration`）按钮：点击后弹出编辑框，分别输入小时和分钟；点击 `完成`（`Done`）关闭编辑框。
4. 删除按钮：内置时长显示灰色垃圾桶，鼠标悬停时提示 `内置时长不可删除`（`Built-in durations cannot be deleted`）；自定义时长的垃圾桶常态为黑色，鼠标悬停时变为红色，删除前显示二次确认。

当前原型展示两个内置时长和一个自定义时长。番茄任务与任务优先级的每一行都使用 28 pt 固定行高，编辑按钮与垃圾桶之间保留清晰的操作间距。

## 番茄任务

### 菜单栏显示

- 菜单栏以“已执行时间 / 总时长”显示任务进度，例如 `10/30min` 表示总时长为 30 分钟、已执行 10 分钟。
- 倒计时结束后，任务名称替换为`已结束`；该文字及进度时间均使用红色粗体显示。
- 菜单栏番茄区域使用固定宽度，不随屏幕或刘海自动变化。`番茄时间`（`Pomodoro Time`）设置提供`菜单栏宽度`（`Menu Bar Width`）滑动条，可调范围为 160–400 pt，默认值为 250 pt；修改即时生效，并持久化至`config.yaml`的`pomodoro.menu_bar_width`。

### 任务详情卡片

- 卡片显示`开始时间`和`预计结束时间`，时间格式仅保留月、日和时间，不显示年份；不再显示“已执行多少分钟”的提示。
- 任务详情默认最多显示三行。内容超过三行时，第三行以省略号截断，并显示`展开`按钮；展开后完整显示内容，卡片高度随内容自适应。点击`收起`可恢复折叠状态。
- 鼠标在菜单栏任务上停留达到`悬停卡片延迟`后自动打开卡片；鼠标离开菜单栏和卡片后，达到同一延迟自动关闭。点击菜单栏任务可立即打开卡片。
- `悬停卡片延迟`以毫秒为单位，范围为 500–3000，默认值为 1000，并持久化至`config.yaml`的`pomodoro.popover_hover_delay_milliseconds`。
- 点击`+15min`或`-15min`后卡片保持显示，便于连续调整。仅当剩余时间不少于 20 分钟时，`-15min`按钮可用。

### 预警规则

- 任务状态标红需同时满足两个条件：剩余时间低于设定分钟数，且剩余比例低于设定百分比。默认阈值为 15 分钟和 20%。
- `番茄时间`设置提供`标红剩余时间`和`标红剩余比例`两个输入项；比例使用直观的百分数输入，例如输入`20`表示低于 20%。对应配置字段为`pomodoro.warning_remaining_minutes`和`pomodoro.warning_remaining_ratio`。
