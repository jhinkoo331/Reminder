## 整体需求说明
本项目是一个类似苹果Reminder的功能，底层代码使用Markdown格式，但是会在Markdown格式的基础上进行修改。

1. 用户启动软件后，首先需要在本地选择一个目录，该目录作为Reminder的工作目录，这样做的目的是便于Reminder的多端同步和备份
2. 在系统设置中添加一个设置项，叫做工作目录。用户可以修改工作目录，并读取该工作目录下的Reminder列表
3. 工作目录下保存多个TXT文件，每个TXT文件对应一个待办事项列表
4. 在工作目录下创建配置文件config.yaml，用于保存系统的所有配置

## 文件解析说明
每一个任务列表的底层是一个TXT文本，每个任务被分为上下两行，上行为JSON，记录该任务的相关信息，下行为text，记录该任务的具体内容。
```json
{
    "CreateTime": "2026-01-01 12:30:00",
    "Deadline": "2026-01-01 12:30:00",
    "Level": 1,
    "Status": "TODO",
}
```
| 字段 | 数据类型 | 说明 |
| ------- | ------- | ------- |
| CreateTime  | Time stamp | 该事项被记录的时间，用户输入时由系统自动创建  |
| Deadline  | Time stamp  | 该事件的截止时间  |
| Level | Int | 大于等于一的整数，Level为一表示该事件为独立事件，Level为二表示该事件为上一个Level为1的事件的子事件，以此类推 |
| Status | 字符串枚举值 | 枚举值包括TODO（待处理）、DONE（已完成）和CANCELED（已取消） |


## 设置
所有设置默认持久化到工作目录中的 `config.yaml`（`config.yaml`）。本章节中，所有会显示在界面上的文字均按“`中文文本`（`English Text`）”标记。

### 窗口整体设计

- 设置窗口标题为 `设置`（`Settings`）。按 `ESC` 关闭设置窗口。
- 设置窗口宽度固定为 520 pt；高度可由用户调整，最小高度为 420 pt。
- 设置内容分为 `通用`（`General`）、`行为`（`Action`）、`任务优先级`（`Priority`）和 `番茄任务`（`Pomodoro Task`）四个 section。
- 语言切换会更新应用内全部系统文案，包括菜单、右键菜单、弹窗、状态名称、错误信息和设置界面；用户创建的任务文本、列表名称及自定义优先级名称不自动翻译。

### `通用`（`General`）

#### 语言与外观

1. `中文/English`（`中文/English`）
   - 使用仅包含两个选项的分段控件，不使用下拉框；左侧固定显示 `中文`（`Chinese`），右侧固定显示 `English`（`English`），不随当前界面语言翻译。
   - 首次启动时根据系统首选语言设置默认值；用户后续选择会持久化到 `config.yaml`（`config.yaml`）。
   - 切换后立即更新应用内所有系统文案。
2. `颜色模式`（`Color Mode`）
   - 使用分段控件，依次显示 `浅色`（`Light`）、`深色`（`Dark`）和 `跟随系统`（`System`）。

#### 工作目录

1. `工作目录`（`Work Directory`）显示当前工作目录路径，路径可选择和复制。
2. 下方提供四个按钮：
   - `选择工作目录`（`Select`）：打开系统文件夹选择器。若已有工作目录，选择新目录后必须显示二次确认；按钮为 `放弃`（`Cancel`）与 `确认`（`Confirm`），其中 `确认`（`Confirm`）使用红色破坏性样式。
   - `打开目录`（`Open`）：在系统文件管理器中打开当前工作目录。
   - `config.yaml`（`config.yaml`）：使用系统默认文本编辑器打开配置文件。
   - `刷新`（`Refresh`）：重新读取工作目录中的列表和配置。
   - 四个按钮的图标与文字使用紧凑间距；`刷新`（`Refresh`）使用与其他按钮一致的线性图标风格。

### `行为`（`Action`）

此 section 位于 `通用`（`General`）之后、`任务优先级`（`Priority`）之前，用于集中展示复制和任务显示行为设置。

1. `显示任务序号`（`Show Task Numbers`）控制任务列表是否显示序号。
2. `复制时包含任务序号`（`Include Task Numbers When Copying`）紧接在其下方并向右缩进，作为 `显示任务序号`（`Show Task Numbers`）的子选项；当父选项关闭时，此选项不可用。
3. `复制时播放音效`（`Play Sound When Copying`）控制复制任务或任务文本时是否播放音效。
4. `完成任务隐藏延迟`（`Completed Task Hide Delay`）使用数值输入框，范围为 0–5000 毫秒；中文界面显示单位 `毫秒`（`Milliseconds`），英文界面显示 `ms`（`ms`）。它控制已完成任务在需要被过滤隐藏前保留在列表中的时长。

### `任务优先级`（`Priority`）

本 section 同时显示系统内置优先级和用户自定义优先级。每一行从左到右包含以下内容：

1. 圆形色块：自定义优先级可点击打开颜色选择器。颜色选择器提供 30 种预设颜色和自定义颜色；内置优先级颜色不可修改。
2. 优先级名称：名称紧邻色块左对齐。内置优先级名称不可修改；自定义优先级名称可修改，且不能为空。若输入为空，立即阻止保存并显示 `名称不能为空`（`Name Is Required`）。
3. 文字样式：`粗体`（`Bold`）、`下划线`（`Underline`）和 `斜体`（`Italic`）。内置和自定义优先级都允许修改这三项。
4. 删除按钮：内置优先级不显示垃圾桶；自定义优先级显示垃圾桶，常态为灰色，鼠标悬停时变为红色。垃圾桶左侧保留两倍于标准控件间距的留白；删除前显示二次确认。
5. `添加优先级`（`Add Priority`）位于全部优先级下方。新行复制当前最下方优先级的颜色和文字样式，名称为 `自定义优先级`（`Custom Priority`）；创建后自动聚焦名称输入框并全选默认名称。

优先级的名称、颜色和文字样式等设置修改均支持 `Command Z`（`Command Z`）撤销上一次修改。

### `番茄任务`（`Pomodoro Task`）

本 section 同时显示内置时长和用户自定义时长。每一行从左到右包含：

1. 计时图标。
2. 名称与时长。时长使用 `__ 小时 __ 分钟`（`__ Hours __ Minutes`）格式的直接数值输入，不使用步进箭头。
3. 删除按钮，仅对自定义时长显示；常态为灰色，鼠标悬停时变为红色，删除前显示二次确认。

番茄任务与任务优先级的每一行使用相同的固定行高。

规则如下：

- 内置时长不允许修改名称或时长，且完全不显示垃圾桶。
- 自定义时长允许修改名称和时长；名称不能为空，空值时立即阻止保存并显示 `名称不能为空`（`Name Is Required`）。
- `添加时间`（`Add Duration`）位于全部时长下方。新时长默认名称为 `30 分钟`（`30 Minutes`），默认时长为 `0 小时 30 分钟`（`0 Hours 30 Minutes`）。
